Content.php : 14

Superclass of all classes that can be added to a page as content.

Any subclass of Content must be declared in a file with the same name as the class and end in .php, ie. Text.php for the Text class. Additionally, the class declaration within that file must match the following regex: /\bclass\s+ClassName\s+extends\s+Content\b/i. In addition to implementing the abstract functions of Content, the subclass should have a constructor that can be called with no parameters. In other words, make sure every parameter of the constructor has a default value specified. Immediately after any object that extends Content is created, you should call Content::set_cms() on it with a reference to the active MeLeeCMS object. Adding such an object to MeLeeCMS with MeLeeCMS::addContent() will do this automatically.

get_properties()

Content.php : 32

Returns the class' important properties that contain the data that users need to define.

public get_properties() : array<string|int, string>

If not overwritten, this function will return an array of each object property except for the internal MeLeeCMS reference. The array is associative, with the keys being the names of the properties and the values being their types (string, boolean, etc). The keys will also be used by __sleep() when the object is serialized. The values will be used by the control panel to determine how users will see the property and define it. Subclasses should overwrite this function in order to provide more specificity with the property types if needed, as well as have better control over which properties are returned, if there are some that need to be excluded.
